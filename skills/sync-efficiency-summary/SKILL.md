# 同步提效总表

## 触发条件
提效记录文档创建完成后，将本次需求数据同步到总文档。

## 需要更新的内容

### 1. 需求级提效记录汇总
在表格末尾追加一行：
- 需求名称、日期、执行工具
- 传统预估工时
- AI 实际工时（待填）
- 提效比（待填）
- AI 参与环节
- 链接到本次单独的提效记录文档

### 2. UI 还原率记录汇总
在表格末尾追加一行：
- 页面/组件名
- 执行工具
- 总 UI 元素数（估算）
- 还原率（待走查后填写）

### 3. 月度汇总表
更新当月的需求数 +1

### 4. 典型 Case（可选）
如果本次需求具有典型性（提效显著 / 新场景 / 有代表性），追加为新 Case。

## 操作方式
通过文档 API 更新对应章节，或告知负责人手动同步。
