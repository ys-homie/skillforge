# 生成实现 Plan

## 触发条件
需求拆解 + Figma 解析完成后，生成实现 Plan 供负责人审核。

## Plan 模板

```markdown
## 实现 Plan: [任务名称]

### 背景
[需求描述，1-3 句话]

### Figma 设计稿
[Figma URL — 实现过程中可随时回溯验证 UI 细节]

### 项目约束
<!-- [项目特定] 列出技术栈约束 -->
- 平台：[iOS / Android / Flutter / RN]
- SDK/语言版本：[具体版本]
- UI 框架：[UIKit / SwiftUI / Compose / XML / Flutter Widget]
- 屏幕适配方案：[AutoLayout / ScreenUtil / dp / 等]
- 状态管理：[项目方案]
- 路由/导航：[项目方案]
- 网络层：[项目方案]
- 国际化：[项目方案，支持语言数]

### 参考现有代码
- 类似页面参考：[具体文件路径]
- 类似组件参考：[具体文件路径]
- 架构写法参考：[具体文件路径]

### UI 规格（从 Figma 提取）
- 布局：[具体描述，优先自适应布局]
- 颜色：[色值 → 项目颜色常量映射，如无则标注需新增]
- 字体：[Figma 字体名 → 项目字体方法/常量映射]
- 间距：[padding/margin 数值]
- 圆角：[数值]

### 切图资源
- [资源名]: [路径] — [格式/倍率说明]
<!-- [项目特定] 按项目资源管理规范填写 -->

### 需要创建的文件
1. [文件路径] — [说明]
2. ...

### 需要修改的文件
1. [文件路径] — [修改内容]
2. ...

### 多语言处理
- **优先复用已有词条！** 添加新 key 前，必须先搜索是否有语义相同或相近的已有词条
- 找到完全匹配 → 直接复用
- 找到语义相近 → 告知负责人决定
- 确认新增 → 所有支持语言同步添加

### 具体实现步骤
1. [第一步：具体描述]
2. [第二步：具体描述]
3. ...

### 埋点要求
<!-- [项目特定] 按项目埋点 SDK 格式填写 -->
- 页面曝光：[埋点方法]
- 按钮点击：[埋点方法]

### 验收标准
- [ ] UI 还原设计稿
- [ ] 功能逻辑正确
- [ ] 多语言 key 已添加（已检查无重复词条）
- [ ] 埋点已添加
- [ ] 无编译警告 / lint 错误
```

## 注意事项
- **Plan 是确认文档**，必须经负责人审核后才能进入代码实现
- Figma 链接必须保留在 Plan 中，便于后续回溯
- 参考代码路径要具体到文件，不要泛泛而谈
- 布局优先自适应，颜色必须映射到项目颜色管理

## 复杂度评估

在 Plan 末尾标注整体复杂度，辅助负责人判断优先级：

| 复杂度 | 判断标准 | 预期 AI 完成度 |
|---|---|---|
| ⭐ 简单 | 纯 UI + 简单逻辑，有现成参考 | 90%+ 一次完成 |
| ⭐⭐ 中等 | 需要新状态管理 + API 对接，有部分参考 | 80%+ 可能需 1-2 轮微调 |
| ⭐⭐⭐ 复杂 | 涉及动画/原生交互/复杂状态，无直接参考 | 60-80% 需多轮迭代 |
