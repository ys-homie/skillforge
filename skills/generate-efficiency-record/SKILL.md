# 生成提效记录

## 触发条件
负责人检查自测通过 + 多语言同步完成（或无需同步）后，负责人明确确认"可以出记录"。

## 模板参考
使用项目的提效跟踪模板，按需求级提效记录表和 UI 还原率记录表的格式填写。

## 文档结构

1. **需求概述** — 需求名称、来源、日期、执行工具、Figma 设计稿链接
2. **需求级提效记录** — 按模板表格格式，填入本次需求数据，`待填` 字段标注
3. **传统工时预估明细** — 按模块拆分的工时基线表
4. **主要变更文件** — 新建文件 + 修改文件清单
5. **UI 还原率记录** — 按模板表格格式，`待走查后填写` 字段标注
6. **归档说明** — 负责人需回填的字段清单

## AI 自动填写的字段

- 需求名称、来源、日期
- 执行工具（Cursor / Codex / 其他）
- Figma 链接
- 传统预估工时（从需求拆解阶段提取）
- AI 参与环节（拆解 / UI还原 / 编码 / Bug修复 / 多语言 / 埋点）
- 变更文件数、组件数
- UI 总元素数（估算）

## 负责人回填字段（标注为待填）

- AI 辅助实际工时（人天）
- UI 还原率（走查后填写）
- Task 链接（仅 Codex / 异步工具执行时）
