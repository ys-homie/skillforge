# 代码实现

## 触发条件
负责人确认 Plan 后，按 Plan 逐步实现代码。

## 执行流程

1. **按 Plan 步骤逐一实现**
2. **实现过程中如有 UI 细节不确定，主动通过 Figma 回查**
3. **完成后输出变更文件清单**

## 通用编码规范

### 错误处理
- 网络请求/异步操作必须有 try-catch + loading/error 状态处理
- 用户可见的错误信息走国际化，日志用英文
- 避免静默吞掉异常，至少输出日志

### 屏幕适配
- 使用项目统一的适配方案，不要硬编码像素值

### 图片资源
<!-- [项目特定] 按项目资源格式填写 -->
- 位图：使用 WebP/PNG，提供多倍率
- 矢量：使用 SVG/PDF/VectorDrawable

### 页面布局
- **自适应优先**：容器尺寸由内容撑开，避免固定死尺寸
- **弹性间距**：使用弹性布局处理间距自适应
- **安全区域**：页面级必须处理安全区域（刘海/底部指示条）

### 颜色
- 优先使用项目颜色常量，不要硬编码色值
- 如果项目颜色管理中没有，先添加再使用

### 字体
- 优先使用项目字体管理方法/常量
- 不要直接硬编码字体样式

## 完成后

输出变更文件清单，**等待负责人检查自测**：
- 负责人进行功能自测、UI 走查、边界场景验证
- 如发现问题，反馈修复，循环直到确认通过

## 提交前自检清单

- [ ] 所有新文件已正确导入，无遗漏依赖
- [ ] 状态管理已正确注册/注入

- [ ] 国际化 key 已在所有支持语言中同步添加
- [ ] 埋点已按需求添加
- [ ] 无编译警告 / lint 错误
